## P2-10：并发运行 → `database is locked` 直接失败

**问题**：两个 `check`（如 watch + 手动check，或 CI 并行）同时跑同一 workspace，会报 `engine init failed: pragma: database is locked` 直接退出。

**定位**：
```bash
grep -n "pragma\|busy_timeout\|database is locked\|sqlite\|SqlitePool\|Connection" engine/src/cli.rs engine/src/**/*.rs | head -20
```

**修复**：对 SQLite 连接设置 `busy_timeout`：
```rust
// 找到 SQLite 连接初始化处，加：
PRAGMA busy_timeout = 5000;
// 或在连接字符串里加 ?timeout=5000
// 或对 "database is locked" 错误做3次重试（间隔500ms）
```

如果使用 `rusqlite`：
```rust
conn.busy_timeout(std::time::Duration::from_secs(5))?;
```

如果使用 `sqlx`：
```rust
// 连接选项里设置
options.busy_timeout(Duration::from_secs(5))
```

**验收**：
```bash
# 同时跑3个 check，都应该完成，不报锁错误
audit-risk check . & audit-risk check . & audit-risk check . & wait
echo "All done, exit codes: $?"
```

## P2-11：`check` 污染 git 工作区

**问题**：单跑 `check`（不先跑 `init`）：
1. 不生成 `.hologram/.gitignore`，导致 `.hologram/` 产物出现在 `git status` 里
2. 把 `.hologram/hologram.db*`、`baseline.json` 计入 `changed_files`，影响分析结果

**定位**：
```bash
grep -n "\.hologram\|gitignore\|changed_files\|git_changed_files" engine/src/cli.rs engine/src/routing/preflight.rs | head -20
```

**修复**：

**Part A**：`check` 第一次写 `.hologram/` 时，也写 `.hologram/.gitignore`：
```rust
fn ensure_hologram_gitignore(workspace: &Path) {
    let gitignore_path = workspace.join(".hologram/.gitignore");
    if !gitignore_path.exists() {
        let _ = fs::write(&gitignore_path, "# auto-generated by audit-risk\n*\n");
    }
}
// 在 check 流程里（build_workspace_check_payload 开头）调用此函数
```

**Part B**：`git_changed_files` 结果里过滤掉 `.hologram/` 自身：
```rust
fn git_changed_files(workspace: &Path) -> Vec<String> {
    // ... 现有逻辑 ...
    // 在返回前加过滤：
    files.retain(|f| !f.starts_with(".hologram/") && f != ".hologram");
    files
}
```

**验收**：
```bash
# 干净仓库，只跑 check（不跑 init）
cd /tmp && git init test-check && cd test-check
echo 'fn main() {}' > main.rs && git add . && git commit -m "init"
audit-risk check .
git status --short
# .hologram/ 不应出现在 git status 里（因为有 .gitignore）
audit-risk check . --json | python3 -c "import sys,json; r=json.load(sys.stdin); files=[f for c in r.get('changed_files',[]) for f in [c]]; print([f for f in files if '.hologram' in f])"
# 输出应为 []（changed_files 里无 .hologram）
```

## P2-12：默认 `check` 对 `require_approval` 级高危仍退出 0

**问题**：默认 `--fail-on = block`，但 high 危风险的 gate decision 是 `require_approval`（低于 block），导致包含如 `os.system(input())` 的代码 CI/hook 绿灯放行。

**定位**：
```bash
grep -n "fail.on\|fail_on\|FailGate\|block\|require_approval" engine/src/cli.rs | head -20
```

**修复（一行改动）**：把默认 `--fail-on` 从 `block` 改为 `require_approval`：
```rust
// 找到 FailGate 的 Default 实现或解析时的默认值
// 改：
impl Default for FailGate {
    fn default() -> Self { FailGate::RequireApproval }  // 改前可能是 Block
}
// 或在 CLI 解析处改默认值
```

同时，在 `init` 生成的 `.githooks/pre-commit` 和 CI 模板里，确认默认也是 `--fail-on require_approval`。

**验收**：
```bash
# 写一个含高危的文件（os.system）
echo 'import os; os.system(input())' > risky.py
git add risky.py
audit-risk check .
echo "Exit code: $?"
# 默认配置下 exit code 应为非0（1或2），不是0
```

## P2-13：基线被篡改/损坏被静默吞掉

**问题**：`baseline.json` 非法/截断时，被静默当"无基线"处理（`unwrap_or_default()`），check 正常运行后再覆盖写回，抹掉了对比基准。

**定位**：`engine/src/routing/preflight.rs` 的 `load_baseline`（约第14行）：
```rust
pub fn load_baseline(workspace: &Path) -> Option<GraphSnapshot> {
    let path = workspace.join(".hologram/baseline.json");
    fs::read_to_string(&path)
        .ok()
        .and_then(|raw| serde_json::from_str(&raw).ok())  // ← 解析失败静默返回 None
}
```

**修复**：区分"文件不存在（正常）"和"文件存在但解析失败（警告）"：
```rust
pub fn load_baseline(workspace: &Path) -> Option<GraphSnapshot> {
    let path = workspace.join(".hologram/baseline.json");
    match fs::read_to_string(&path) {
        Err(_) => None,  // 文件不存在，正常
        Ok(raw) => match serde_json::from_str::<GraphSnapshot>(&raw) {
            Ok(baseline) => Some(baseline),
            Err(error) => {
                // 文件存在但解析失败 → 告警，不静默
                eprintln!(
                    "警告：.hologram/baseline.json 损坏或格式不兼容（{error}），\
                     本次以无基线模式运行。如需重置，删除该文件后重新运行 check。"
                );
                None
            }
        },
    }
}
```

**验收**：
```bash
# 损坏基线
echo '{bad json' > .hologram/baseline.json
audit-risk check . 2>&1 | grep -i "基线\|baseline\|损坏\|corrupt"
# 应输出警告，而不是静默通过
```
