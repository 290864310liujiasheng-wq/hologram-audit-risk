# 产品章程

## 一句话

AI 编码风控平台：以通用 CLI `audit-risk` 为主交付形态，在开发者使用 AI 写代码的过程中，持续审查生成代码和变更 diff，发现风险、用白话逐行解释、按规则拦截危险操作，并留下可追溯审计证据。

## 目标客户

- 已经在团队内使用 AI 编码工具的研发团队。
- 需要控制 AI 生成代码质量、安全、权限、架构边界和合规留痕的组织。
- 希望接入自有模型 API、内网模型或合规网关，而不是把代码和 key 交给平台统一转发的客户。

## 核心承诺

- 开发不中断：审查尽量旁路、增量、可取消，不阻塞普通阅读和编辑。
- 风险可解释：每个 finding 必须说明位置、原因、影响、建议处理方式，面向普通工程师也能读懂。
- 规则可拦截：高风险命中可以阻止工具调用、提交、修复或发布动作。
- 审计可追溯：谁触发、哪个模型、哪个规则、读了哪些证据、给了什么决策，都应形成 append-only 记录。
- 接入可控：客户配置自己的模型 provider、base URL、模型名和 key；平台不托管统一模型额度。
- 工具通用：平台必须能被 Codex、Cursor、Copilot、CodeGeeX 等外部 AI 编码工具通过 CLI、Hook 或 CI 调用。

## 第一版范围

- 工作区级代码审查任务。
- AI 生成/修改代码的 diff 审查。
- 规则命中、白话解释、风险等级、拦截建议。
- 本地权限审批与审计留痕。
- 多 provider 接入的合同边界。
- 为多代理并行审计和受控自修复预留结构。
- 桌面管理后台，用于规则配置、审计检索和导出报告。

## 非目标

- 不做通用聊天机器人。
- 不做纯代码可视化玩具。
- 不做模型 API 转售平台。
- 不承诺自动修复所有问题。
- 不绕过客户现有代码托管、CI、权限和审计流程。
- 不再把独立桌面 IDE 当作主产品形态。
- 不在第一版实现完整企业权限、云端多租户和集中策略市场，除非用户后续明确改变优先级。

## 现有基座映射

- CLI、代码图谱与静态分析：`engine/`。
- 桌面管理后台、本地能力、凭证、审计、沙箱：`src-tauri/src/`。
- 模型 provider 抽象：`src-ui/src/provider/`。
- Agent 工具循环和权限门禁：`src-ui/src/agent/`。
- 深色管理后台 UI：`src-ui/src/ui/`。

## 成功标准

- 后续 agent 能从本文档直接判断：该做 CLI-first 的风控平台，不该继续泛化成 HoloGram 展示工具或桌面 IDE 主产品。
- 任一新功能都能落到 `architecture.md` 的 owner 层和 `contracts.md` 的合同。
- 任一风险结论都能关联代码位置、证据、规则、模型输出和审计事件。

## 停止条件

第一版真源文档完成的条件：

- `AGENTS.md` 明确当前主线和 agent 行为边界。
- `dev-docs/README.md` 建立 active docs 索引。
- `product-charter.md`、`architecture.md`、`domain-model.md`、`contracts.md` 相互一致。
- 旧 HoloGram 语义被限定为现有基座或历史外部说明，不再覆盖当前产品边界。
